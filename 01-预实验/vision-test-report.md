## 6. 关键发现与问题

### 6.1 优势

1. **数字类信息极其准确**：角色、回合数、能量、血量、金币等纯数字/图标信息识别率接近100%。
2. **场景感知能力强**：能准确判断当前是战斗、商店、地图、事件还是角色选择界面，不会 hallucinate 不存在的战斗信息。
3. **卡牌名称识别尚可**：对常见卡牌（Strike、Defend、Bash 等）能正确识别。

### 6.2 劣势与风险

1. **卡牌效果高度依赖知识推断，而非OCR读取**：
   - 模型对"Strike 造成6点伤害"的回答，更多是基于对游戏的知识记忆，而非从截图文字中读取。
   - 当遇到Mod卡牌、新卡牌或截图分辨率较低时，效果描述会不可靠。
   - **这是最大技术风险**——策略建议高度依赖"卡牌效果"的准确性，如果读错效果，策略会完全错误。

2. **BOSS和稀有敌人识别不稳定**：
   - 截图07中模型对BOSS名称表现出明显的不确定性，先后给出多个猜测。
   - 对于策略建议而言，"知道打谁"和"知道敌人的意图"同样重要。

3. **手牌数量偶发错误**：
   - 截图06中模型标注"4张"却列出5张卡牌，说明数量统计存在疏忽。

4. **版本/Mod干扰**：
   - 测试截图显示 `v0.98.1` 版本，部分卡牌（如 Juggling、Breakthrough、Ashen Strike）疑似为 Mod 或早期测试版内容，正式版 STS2 中可能不存在。模型对这些非标准卡牌的识别能力未经验证。

## 7. 核心结论

### 可行性判断：**有条件可行，但存在关键风险**

| 场景 | 可行性 | 说明 |
|------|--------|------|
| **基础状态展示**（当前角色、血量、能量、回合） | ✅ **可行** | 识别精度高，可直接用于UI展示 |
| **场景类型判断**（战斗/商店/地图/事件） | ✅ **可行** | 准确率高，可用于路由不同策略模块 |
| **敌人身份识别（常见小怪）** | ⚠️ **基本可行** | 对标准敌人准确率尚可 |
| **敌人身份识别（BOSS/精英/稀有）** | ❌ **风险较高** | 识别不稳定，建议要求用户确认 |
| **手牌名称识别** | ⚠️ **基本可行** | 常见卡牌可识别，但需校验 |
| **手牌效果识别** | ❌ **不可直接依赖** | 主要依赖知识推断，非图像OCR，**这是最大短板** |
| **给出策略建议** | ⚠️ **可用但需兜底** | 在常见场景下可用，但必须增加用户确认/纠错机制 |

### 一句话结论

> **Kimi Vision 能可靠读取"基础数字状态"（角色、血量、能量、回合），但无法可靠读取"策略关键信息"（卡牌效果、BOSS身份）。直接用于全自动策略建议存在风险，需要增加确认/纠错环节或配合其他技术方案。**

## 8. 替代方案建议

如果产品要求更高的策略可靠性，推荐以下替代或增强方案：

### 方案A：人机确认流程（推荐，成本最低）

```
用户截图 → AI识别 → 展示识别结果供用户确认/修正 → 生成策略建议
```

- 用户可一键纠正角色、敌人名称、卡牌效果等关键信息
- 适用于 MVP 阶段，在识别精度不足时保证策略质量

### 方案B：OCR + 规则引擎增强（推荐，精度最高）

```
用户截图 → 传统OCR提取卡牌文字 → 与卡牌数据库匹配 → 补充Vision识别结果 → 生成策略建议
```

- 对卡牌区域做专门的文字OCR，不依赖大模型的"记忆推断"
- 建立杀戮尖塔卡牌数据库（名称→效果映射），OCR只负责"读到名称"
- 敌人识别也可通过模板匹配/UI元素定位来增强

### 方案C：游戏内存读取（仅限PC端，精度最高但限制大）

```
用户授权 → 读取游戏进程内存 → 直接获取结构化游戏状态 → 生成策略建议
```

- 100%准确，不受截图质量影响
- 但仅适用于PC端，且需要用户安装辅助工具，移动端不可行
- 技术门槛高，可能触及游戏反作弊机制

### 方案D：多模态 + 知识库混合（中长期方案）

```
用户截图 → Vision提取视觉特征 → 结构化知识库查询（卡牌/敌人/遗物） → 融合生成策略
```

- 用Vision做"定位"（找出卡牌区域、敌人头像），用数据库做"识别"（匹配具体卡牌/敌人）
- 需要构建和维护STS2的完整游戏数据知识库

## 9. 给产品的建议

| 阶段 | 建议方案 | 预期准确率 |
|------|----------|------------|
| **MVP** | Vision + 人机确认 | 基础状态100%，策略建议经用户确认后100% |
| **V1.0** | Vision + OCR卡牌文字 + 卡牌数据库 | 卡牌效果准确率提升至90%+ |
| **V2.0** | Vision + 完整游戏知识库 + 模板匹配 | 整体识别准确率95%+ |

**关键决策点：** 是否接受"AI先给建议，用户负责验证"的交互模式？如果可以接受，Kimi Vision 足以支撑MVP；如果要求全自动零确认，当前精度不够。
