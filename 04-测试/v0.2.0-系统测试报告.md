## 三、数据库核查结果

```
archetypes 表字段（实际）：
  id, game_version, character, name_cn, name_en,
  description, core_cards, support_cards, key_relics,
  avoid_cards, early_game, mid_game, late_game
  → 共13个字段

archetypes 表字段（方案要求）：
  以上13个 + difficulty_rating, win_rate, recommended_ascension,
  playstyle_tags, mvp_relics, avoid_relics, key_nodes,
  advanced_tips, route_preferences, data_source, updated_at
  → 共23个字段

archetype_attempts 表：❌ 不存在
archetypes 记录数：0（种子数据未灌入）
cards 记录数：295
game_sessions 记录数：4
```

**结论**：生产数据库是用 **旧 schema** 创建的，新 schema 定义虽然存在于 schema.sql 文件中，但 **从未执行 ALTER TABLE** 或重建。种子数据文件 `archetypes.ts` 虽存在但从未灌入。

## 四、Bug 实测发现

### Bug 1：卡牌翻译严重错误（P0）

实测 API `/api/cards` 返回数据中：

| id | name_en | name_cn（实际） | name_cn（应为） |
|----|---------|----------------|----------------|
| 231 | Swift Strike | **科学方法** | **迅捷打击** |
| 245 | Madness | **madness**（小写英文） | **疯狂** |
| 232 | Bandage Up | ** Bandage Up**（带前导空格） | **绷带** |
| 233 | Blind | ** blind**（小写带空格） | **致盲** |
| 235 | Deep Breath | ** deep breath** | **深呼吸** |

> 大量 colorless 卡牌的中文名是英文或带前导空格，严重影响搜索和展示。

### Bug 2：AI 错误处理粗糙

当前降级只有简单的 try/catch，无错误分类：
- 401 → API_KEY_INVALID（可恢复，提示检查密钥）
- 429 → RATE_LIMITED（可恢复，提示稍后再试）
- fetch failed → NETWORK_ERROR（可恢复）
- timeout → TIMEOUT（可恢复）

### Bug 3：AI 回复无免责声明

Mock 和真实 API 返回的策略建议均未附加免责声明。
