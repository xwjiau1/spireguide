## 一、版本定位与目标

v0.2.0 是 SpireGuide 的**核心功能迭代版本**，以「流派指南」为核心卖点，同时完善 AI 助手真实可用性，修复 v0.1.x 测试阶段暴露的关键 Bug。

### 1.1 功能优先级（P0/P1/P2）

| 优先级 | 模块 | 功能项 | 说明 |
|--------|------|--------|------|
| **P0** | 流派指南 | 流派列表 + 详情页 | 核心卖点，必须完整可用 |
| **P0** | 流派指南 | 尝试与心得记录 | 用户价值闭环 |
| **P0** | AI 助手 | 接入 Kimi API + 降级处理 | 从 Mock 升级到真实可用 |
| **P0** | Bug 修复 | 中文编码乱码 | 严重影响体验 |
| **P0** | Bug 修复 | 卡牌翻译修正 | 数据准确性 |
| **P1** | AI 助手 | 结合流派攻略给建议 | 差异化竞争力 |
| **P1** | Bug 修复 | StS2 卡牌数据补充 | 数据完整性 |
| **P1** | 流派指南 | 路线推荐（融入流派页） | PRD 原需求，轻量实现 |
| **P2** | 其他 | 种子查询功能 | 可延后或融入尝试记录 |

## 二、数据库 Schema 变更（DDL）

### 2.1 现有 Schema 基线

当前数据库已包含以下表：
- `cards` — 295 张卡牌（StS1 + StS2 部分）
- `relics` — 遗物数据
- `enemies` — 敌人/BOSS 数据
- `game_sessions` — 对局记录
- `session_screenshots` — 截图记录
- `qa_history` — AI 问答历史
- `ai_configs` — AI 配置（单条记录）
- `card_combos` — 卡牌配合
- `archetypes` — **流派表（已存在但为空）**
- `card_archetype_links` — 卡牌流派关联

### 2.2 新增/修改表

#### 表 1：`archetypes` 扩充字段（已存在表 ALTER）

当前 `archetypes` 表结构较简单，需要扩展字段以支持完整流派攻略：

```sql
-- 流派表字段扩充（兼容已有表，使用 ALTER TABLE）
ALTER TABLE archetypes ADD COLUMN difficulty_rating INTEGER DEFAULT 3;      -- 难度评分 1-5
ALTER TABLE archetypes ADD COLUMN win_rate REAL;                               -- 预估胜率（如 0.35）
ALTER TABLE archetypes ADD COLUMN recommended_ascension INTEGER DEFAULT 0;    -- 推荐起始进阶
ALTER TABLE archetypes ADD COLUMN playstyle_tags TEXT;                         -- JSON ["aggro","combo","defensive"]
ALTER TABLE archetypes ADD COLUMN mvp_relics TEXT;                             -- 核心遗物 JSON [{"id":1,"reason":"..."}]
ALTER TABLE archetypes ADD COLUMN avoid_relics TEXT;                           -- 避免遗物 JSON
ALTER TABLE archetypes ADD COLUMN key_nodes TEXT;                              -- 关键节点说明（长文本）
ALTER TABLE archetypes ADD COLUMN advanced_tips TEXT;                          -- 进阶技巧（长文本）
ALTER TABLE archetypes ADD COLUMN route_preferences TEXT;                      -- 路线偏好 JSON {"early":"精英","mid":"商店","late":"休息"}
ALTER TABLE archetypes ADD COLUMN data_source TEXT DEFAULT 'manual';           -- 数据来源：manual|scraped|hybrid
ALTER TABLE archetypes ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP;

-- 为已有字段增加索引
CREATE INDEX IF NOT EXISTS idx_archetypes_character ON archetypes(character);
CREATE INDEX IF NOT EXISTS idx_archetypes_game ON archetypes(game_version);
```

#### 表 2：`archetype_attempts` — 流派尝试记录（新建）

```sql
-- 流派尝试与心得记录表
CREATE TABLE IF NOT EXISTS archetype_attempts (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    archetype_id    INTEGER NOT NULL,
    device_id       TEXT,                       -- 匿名用户标识
    character       TEXT NOT NULL,              -- 实际使用的角色
    seed            TEXT,                       -- 游戏种子
    ascension       INTEGER DEFAULT 0,          -- 进阶等级
    outcome         TEXT,                       -- 'win'|'lose'|'abandoned'|'in_progress'
    final_floor     INTEGER,
    duration_minutes INTEGER,                   -- 对局时长（分钟）
    notes           TEXT,                       -- 心得体会（长文本，用户自由输入）
    rating          INTEGER CHECK(rating BETWEEN 1 AND 5), -- 用户评分
    is_favorite     INTEGER DEFAULT 0,          -- 0|1 是否收藏
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (archetype_id) REFERENCES archetypes(id) ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS idx_attempts_archetype ON archetype_attempts(archetype_id);
CREATE INDEX IF NOT EXISTS idx_attempts_device ON archetype_attempts(device_id);
CREATE INDEX IF NOT EXISTS idx_attempts_created ON archetype_attempts(created_at);
```

#### 表 3：`archetype_guides` — 流派攻略内容扩展表（新建，可选方案）

> **决策说明**：如果 `archetypes` 表扩充后的字段足够存储所有攻略内容，则不需要此表。但如果攻略内容非常丰富（每个流派 2000+ 字），建议拆分存储。

```sql
-- 流派攻略详细内容表（如果内容量极大则启用）
CREATE TABLE IF NOT EXISTS archetype_guides (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    archetype_id    INTEGER NOT NULL UNIQUE,     -- 一对一
    full_guide_md   TEXT,                       -- Markdown 格式完整攻略
    card_priority   TEXT,                       -- 卡牌抓取优先级 JSON
    potion_priority TEXT,                       -- 药水优先级
    event_choices   TEXT,                       -- 关键事件选择
    boss_strategies TEXT,                       -- 各 BOSS 对策
    FOREIGN KEY (archetype_id) REFERENCES archetypes(id) ON DELETE CASCADE
);
```

**Irra 建议**：v0.2.0 阶段直接将攻略内容存放在 `archetypes` 表的 `early_game`/`mid_game`/`late_game`/`key_nodes`/`advanced_tips` 字段中，避免过度拆分。如果后续内容膨胀再考虑分表。

### 4.4 流派详情查询的 JOIN 优化

```sql
-- GET /api/archetypes/:id 的核心查询
-- 流派基本信息
SELECT * FROM archetypes WHERE id = ?;

-- 核心卡牌详情（通过 card_archetype_links JOIN cards）
SELECT c.*, cal.role 
FROM cards c
JOIN card_archetype_links cal ON c.id = cal.card_id
WHERE cal.archetype_id = ? AND cal.role = 'core'
ORDER BY c.cost, c.name_cn;

-- 关键遗物（通过 archetypes.mvp_relics 中的 ID 查询 relics 表）
-- 或者新增 archetype_relic_links 表做正规关联

-- 尝试记录
SELECT * FROM archetype_attempts 
WHERE archetype_id = ? AND (device_id = ? OR device_id IS NULL)
ORDER BY created_at DESC;
```

## 五、数据抓取策略

### 5.1 流派数据来源分析

| 来源 | 可靠性 | 抓取难度 | 数据质量 | 法律风险 |
|------|--------|----------|----------|----------|
| Reddit (r/slaythespire) | 中 | 低 | 中 | 低 |
| Steam 社区指南 | 高 | 中 | 高 | 低 |
| 中文社区（贴吧/B站/小黑盒） | 中 | 中 | 中高 | 低 |
| 游戏 Wiki (Fandom) | 高 | 低 | 高 | 低 |
| 官方 Patch Notes | 高 | 低 | 极高 | 无 |
| 专业攻略站（如 SpireLogs） | 高 | 中 | 极高 | 低 |
| YouTube 攻略视频 | 中 | 高 | 中 | 低 |

### 5.2 推荐抓取方案

**Irra 建议采用「混合策略」**：

```
┌─────────────────────────────────────────────┐
│              流派数据来源架构                 │
├─────────────────────────────────────────────┤
│                                             │
│  第一层：人工录入基础数据（MVP 阶段）          │
│  ├── 每个角色 3-5 个主流流派                  │
│  ├── 共 6 角色 × 4 流派 = ~24 条记录         │
│  └── 工作量：1-2 天（参考 Wiki + 社区经验）   │

│  └── 频率：每季度一次或随版本更新             │
│                                             │
│  第三层：社区数据抓取（自动化）                │
│  ├── 目标：Reddit 每周讨论帖、Steam 指南      │
│  ├── 内容：新兴流派、胜率数据、进阶技巧       │
```

## 六、Bug 修复技术方案

### 6.1 Bug 1：中文编码乱码

#### 问题诊断

**现象**：API 返回中文显示为乱码（如 `å‡»` 代替 `打击`）。

**根因分析**（可能性排序）：
1. **HTTP 响应头缺少 `charset=utf-8`** — Express 默认发送 `Content-Type: application/json`，可能不带 charset
2. **better-sqlite3 读取时编码问题** — SQLite 数据库编码设置或读取时未正确处理 UTF-8
3. **前端 fetch 解析问题** — `res.json()` 默认按 UTF-8 解析，但如果响应头错误，可能按其他编码解析

#### 修复方案

**步骤 1：强制 Express 响应 UTF-8**

```typescript
// backend/src/index.ts
// 在现有中间件之后，路由之前添加
app.use((req, res, next) => {
  res.setHeader('Content-Type', 'application/json; charset=utf-8');
  next();
});
```

> **注意**：这会影响所有响应。如果某些响应不需要 JSON，可以改为在路由层面处理。

更好的方案——全局中间件：

```typescript
// 新增 middleware/charset.ts
import { Request, Response, NextFunction } from 'express';

export function charsetMiddleware(req: Request, res: Response, next: NextFunction) {
  const originalJson = res.json.bind(res);
  res.json = function(body: any) {
    res.setHeader('Content-Type', 'application/json; charset=utf-8');
    return originalJson(body);
  };
  next();
}
```

**步骤 2：确保数据库编码正确**

**步骤 3：前端 fetch 防御性处理**

```typescript
// frontend/src/services/api.ts
async function fetchApi<T>(url: string, options?: RequestInit): Promise<T> {
  const res = await fetch(`${API_BASE}${url}`, {
    headers: {
      'Content-Type': 'application/json; charset=utf-8',
      'Accept': 'application/json; charset=utf-8',
    },
    ...options,
  });
  
  // 防御性：如果响应头没有 charset，强制按 UTF-8 解析
  const contentType = res.headers.get('content-type') || '';
  if (!contentType.includes('charset')) {
    // 手动读取并解析
    const buffer = await res.arrayBuffer();
    const text = new TextDecoder('utf-8').decode(buffer);
    const data = JSON.parse(text);
    if (!data.success) {
      throw new Error(data.error || '请求失败');
    }
    return data;
  }
  
  const data = await res.json();
  if (!data.success) {
    throw new Error(data.error || '请求失败');
  }
  return data;
}
```

```typescript
function validateCardData(cards: any[]) {
  const issues: string[] = [];
  
  for (const card of cards) {
    // 必填字段检查
    if (!card.name_cn || !card.name_en) {
      issues.push(`缺失名称: ${JSON.stringify(card)}`);
    }
    // 费用范围检查
    if (card.cost < -1 || card.cost > 10) {
      issues.push(`费用异常: ${card.name_cn} = ${card.cost}`);
    }
    // 类型有效性检查
    const validTypes = ['attack', 'skill', 'power', 'curse', 'status'];
    if (!validTypes.includes(card.type)) {
      issues.push(`类型无效: ${card.name_cn} = ${card.type}`);
    }
    // 稀有度检查
    const validRarities = ['basic', 'common', 'uncommon', 'rare', 'special'];
    if (!validRarities.includes(card.rarity)) {
      issues.push(`稀有度无效: ${card.name_cn} = ${card.rarity}`);
    }
  }
  
  return issues;
}
```

```typescript
// backend/src/routes/ai.ts — 优化后的错误处理

// 定义错误类型
enum AiErrorType {
  NETWORK_ERROR = 'network_error',        // 网络不通
  API_KEY_INVALID = 'api_key_invalid',    // 密钥错误
  RATE_LIMITED = 'rate_limited',          // 触发限流
  TIMEOUT = 'timeout',                    // 超时
  MODEL_ERROR = 'model_error',            // 模型内部错误
  UNKNOWN = 'unknown',
}

interface AiError {
  type: AiErrorType;
  message: string;
  recoverable: boolean;  // 是否可降级到 Mock
}

function classifyError(err: any): AiError {
  const msg = err.message || '';
  if (msg.includes('401') || msg.includes('Unauthorized')) {
    return { type: AiErrorType.API_KEY_INVALID, message: 'API密钥无效或已过期', recoverable: true };
  }
  if (msg.includes('429') || msg.includes('Too Many Requests')) {
    return { type: AiErrorType.RATE_LIMITED, message: '请求过于频繁，请稍后再试', recoverable: true };
  }
  if (msg.includes('fetch failed') || msg.includes('ECONNREFUSED')) {
    return { type: AiErrorType.NETWORK_ERROR, message: '网络连接失败，请检查网络', recoverable: true };
  }
  if (msg.includes('timeout')) {
    return { type: AiErrorType.TIMEOUT, message: '请求超时', recoverable: true };
  }
  return { type: AiErrorType.UNKNOWN, message: `未知错误: ${msg}`, recoverable: true };
}

// 在 recognize 和 strategy 路由中使用
async function callAiWithFallback<T>(
  realCall: () => Promise<T>,
  mockCall: () => T,
  res: Response
): Promise<T> {
  try {
    return await realCall();
  } catch (err) {
    const aiErr = classifyError(err);
    console.warn(`[AI] ${aiErr.type}: ${aiErr.message}`);
    
    if (aiErr.recoverable) {
      // 返回降级响应，但标记为 mock
      const mockResult = mockCall();
      (mockResult as any)._fallback = true;
      (mockResult as any)._error = aiErr.message;
      return mockResult;
    }
    throw err;
  }
}
```

```sql
    rating          INTEGER CHECK(rating BETWEEN 1 AND 5),
    is_favorite     INTEGER DEFAULT 0,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (archetype_id) REFERENCES archetypes(id) ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS idx_attempts_archetype ON archetype_attempts(archetype_id);
CREATE INDEX IF NOT EXISTS idx_attempts_device ON archetype_attempts(device_id);
CREATE INDEX IF NOT EXISTS idx_attempts_created ON archetype_attempts(created_at);
```
