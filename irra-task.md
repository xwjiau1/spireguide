## 一、项目背景

创始人JiaWen要求开发一款杀戮尖塔（Slay the Spire）AI攻略平台。

**核心功能：**
- 玩家截图游戏画面 → AI识别状态 → 给出策略建议
- 卡牌/遗物/敌人数据库浏览
- 对局截图记录
- 支持StS1和StS2两代游戏

**技术验证结论（Irra已完成预实验）：**
- Kimi Vision能可靠识别角色/血量/能量/回合（~98%）
- 手牌名称识别~90%，但手牌效果识别仅~40%（最大风险）
- MVP方案：Vision + 人机确认流程（AI识别后用户确认/修正，再出策略）

## 三、开发范围（MVP v0.1）

### P0功能（必须全部完成）

1. **AI截图问答**（核心差异化功能）
   - 截图上传（拖拽/粘贴）
   - Kimi Vision识别游戏状态
   - 人机确认流程：展示识别结果让用户确认/修正
   - 确认后调用Kimi Chat生成策略建议
   - 支持StS1和StS2

2. **卡牌/遗物即时查询**
   - 按名称（中英文）、费用、类型、角色筛选
   - 显示卡牌完整信息+升级后效果
   - 显示常见配合、适用流派
   - 支持两代数据切换

3. **卡牌数据库**
   - StS1：4角色全部卡牌（~350张）
   - StS2：Early Access已有角色卡牌
   - 浏览、搜索、筛选、对比
   - 数据随版本更新

4. **敌人/BOSS数据库**
   - StS1和StS2两代敌人数据
   - 血量、意图模式、技能、推荐对策

5. **对局截图记录**
   - 关键节点手动截图保存
   - 时间线展示
   - 文字备注
   - 本地存储优先

### P1/P2功能（不做，预留接口）

- 路线选择建议
- 卡组流派分析
- 遗物组合推荐
- 网络攻略自动同步
- AI自动复盘
- 联机模式策略
- 数据统计看板
- 社区功能

## 四、技术约束

1. **技术栈（按架构方案）：**
   - 前端：React 18 + TypeScript + Tailwind CSS
   - 后端：Node.js（Express或NestJS）
   - 数据库：SQLite（MVP阶段，轻量优先）
   - AI：Kimi API（视觉+文本）
   - 部署：Vercel / 自有服务器

2. **数据完整性：**
   - 卡牌数据库必须兼容StS1和StS2
   - 预留`game_version`字段
   - 卡牌Schema预留扩展字段（StS2持续更新）

3. **AI幻觉缓解：**
   - 必须实现"人机确认流程"
   - 识别结果展示给用户，用户可修正
   - 确认后再调用AI生成策略
   - 策略回答中标注"仅供参考"

4. **代码质量：**
   - 中文注释
   - 模块化结构
   - 关键逻辑有说明
   - 错误处理完善

5. **MVP优先：**
   - 只做P0功能
   - 避免过重依赖
   - 保持包体小
