## 新增功能：AI系统设置

响应CEO需求：将AI配置作为系统配置，通过可视化方式开启，支持根据AI协议配置Key和URL。

---

### 一、后端变更

| 文件 | 变更 | 说明 |
|------|------|------|
| `database/schema.sql` | + ai_configs表 | 存储AI配置（开关、协议、密钥、模型参数） |
| `routes/config.ts` | 新增 | 配置API：获取/更新/测试AI连接 |
| `routes/ai.ts` | 重写 | 支持真实API调用（配置开启时），否则Mock回退 |
| `index.ts` | + config路由注册 | `/api/config` |

**AI配置表字段：**
- `enabled` — 0=关闭 1=开启
- `provider` — kimi / openai / custom
- `api_key` — API密钥（前端安全显示）
- `api_url` — 自定义地址（可选）
- `model` / `vision_model` — 策略/视觉模型
- `timeout_ms` / `max_tokens` / `temperature` — 高级参数
- `system_prompt` — 系统提示词

**AI路由逻辑：**
1. 读取配置 → 检查enabled和api_key
2. 若开启且有密钥 → 调用真实API（Kimi/OpenAI/自定义）
3. 若关闭或调用失败 → Mock回退（启发式策略 + 数据库拼接）
4. 策略回答自动标注来源（real-api / mock）

---

### 二、前端变更

| 文件 | 变更 | 说明 |
|------|------|------|
| `pages/SettingsPage.tsx` | 新增 | AI设置页面（6个区块） |
| `App.tsx` | + 导航/路由 | 侧边栏新增「系统设置」入口 |
| `services/api.ts` | + configApi | 配置API客户端 |

**设置页面功能：**
1. **AI开关** — 一键开启/关闭，关闭时显示Mock提示
2. **提供商选择** — Kimi / OpenAI / 自定义（切换时自动填充默认值）
3. **API密钥** — 密码输入框，支持显示/隐藏，安全显示已保存密钥
4. **API地址** — 自定义URL（可选，默认使用提供商地址）
5. **模型配置** — 策略模型 + 视觉模型
6. **高级参数** — 超时、最大Token、温度滑块、系统提示词
7. **测试连接** — 验证API密钥有效性，显示模型和响应
8. **保存配置** — 持久化到SQLite

### 四、使用方式

1. 启动后端服务
2. 打开前端 → 侧边栏「系统设置」
3. 开启AI开关
4. 选择提供商（Kimi/OpenAI/自定义）
5. 输入API密钥
6. 点击「测试连接」验证
7. 点击「保存配置」
8. 返回「AI助手」页面，截图识别和策略建议将调用真实AI
