## 关键决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| SQLite vs PostgreSQL | SQLite | MVP轻量部署，单文件便于备份 |
| better-sqlite3版本 | v12.1.0 | 兼容Node 24（v9编译失败） |
| AI实现方式 | Mock (kimi-mock) | MVP阶段先验证流程，部署前替换为真实API |
| 卡牌收录策略 | 代表性收录(~295张) | 各核心角色~40张+无色/诅咒，StS2按EA内容 |
| 匿名使用 | device_id | 无需登录，降低使用门槛 |
| 图片存储 | 本地文件系统 | MVP简单可靠，后续可迁移对象存储 |
| 前端路由 | HashRouter备选 | 当前BrowserRouter + SPA回退，Nginx配置时注意 |

## 踩坑记录

1. **better-sqlite3编译失败**
   - 现象：npm install 时 node-gyp 编译报错（V8 API不兼容）
   - 根因：v9.4.3 不支持 Node 24 的 V12 V8 API
   - 解决：升级至 v12.1.0

2. **ts-node类型声明缺失**
   - 现象：`seed.ts` 运行时报 `TS7016: Could not find declaration file`
   - 解决：安装 `@types/better-sqlite3`

3. **Express路由顺序陷阱**
   - 现象：`/api/cards/compare` 返回404
   - 根因：`router.get('/:id')` 定义在 `/compare` 之前，`/compare` 被匹配为 `id='compare'`
   - 解决：特定路由必须放在参数化路由之前

4. **静态文件路径dev/prod不一致**
   - 现象：开发时 `__dirname` = `src/backend/src/`，产物在 `src/backend/public`
   - 解决：运行时探测多个可能路径，取第一个存在的
