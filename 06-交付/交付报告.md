## v0.1.1 更新摘要（本次新增）

### AI系统配置可视化

响应CEO需求，新增可视化AI配置面板，支持通过界面而非环境变量管理AI参数。

**后端变更：**
- `database/schema.sql` — 新增 `ai_configs` 表（单条记录，开关/协议/密钥/模型参数）
- `routes/config.ts` — 新增配置API：`GET/POST /api/config/ai`，测试连接 `/api/config/ai/test`
- `routes/ai.ts` — 重构为支持真实API调用（Kimi/OpenAI/自定义），失败自动降级Mock
- `index.ts` — 注册 `/api/config` 路由

**前端变更：**
- `pages/SettingsPage.tsx` — 新增6区块设置页面（AI开关、提供商、API密钥、模型、高级参数、连接测试）
- `App.tsx` — 侧边栏新增「系统设置」入口
- `services/api.ts` — 新增 `configApi` 封装

**安全设计：**
- 密钥仅保存在本地SQLite，不上传到任何第三方
- 前端显示掩码版本（前4位 + **** + 后4位），通过 `has_key` 布尔值提示
- 测试连接端点复用标准 `/chat/completions` 接口验证连通性

## 五、部署方案摘要

- **方式一（推荐演示）：** 进入 `06-交付/build/`，`npm install --production && bash start.sh`
- **方式二（Docker）：** 提供 Dockerfile，单容器部署
- **方式三（PM2守护）：** `pm2 start dist/index.js`
- **反向代理：** 附Nginx配置模板
- **数据备份：** SQLite单文件热复制即可

## 六、已知限制与风险点

| 风险 | 级别 | 说明 | 缓解措施 |
|------|------|------|----------|
| AI为Mock实现 | 🟡 中 | 策略基于启发式规则+数据库拼接，准确率有限 | 部署前接入真实Kimi API（代码已预留接入点） |
| StS2数据不完整 | 🟡 中 | Early Access阶段，仅收录已知内容 | EA发售后更新种子数据 |
| 单进程部署 | 🟢 低 | 未做集群适配，高并发可能瓶颈 | MVP阶段单用户场景足够 |
| 本地文件存储 | 🟢 低 | 截图存在本地磁盘，多节点部署需对象存储 | 后续可迁移至S3/OSS |
| 无用户认证 | 🟢 低 | 匿名使用，数据隔离靠device_id | 如需用户系统后续可叠加 |
