# SpireGuide 部署方案

## 环境要求

| 组件 | 最低版本 | 说明 |
|------|----------|------|
| Node.js | 20.x | 推荐 24.x |
| npm | 10.x | 随 Node.js 附带 |
| 内存 | 512MB | 数据库+后端+前端静态文件 |
| 磁盘 | 100MB | 含数据库和上传文件预留 |

## 部署方式

### 方式一：一键启动（推荐，用于演示/个人使用）

```bash
# 1. 进入构建产物目录
cd 06-交付/build

# 2. 安装生产依赖
npm install --production

# 3. 启动服务
bash start.sh
```

服务将运行在 `http://localhost:3001`

### 方式二：Docker 部署（推荐，用于生产）

```dockerfile
FROM node:24-alpine
WORKDIR /app
COPY dist ./dist
COPY package.json .
RUN npm install --production
ENV NODE_ENV=production
ENV PORT=3001
EXPOSE 3001
CMD ["node", "dist/index.js"]
```

构建并运行：
```bash
docker build -t spireguide .
docker run -p 3001:3001 -v $(pwd)/data:/app/dist/data -v $(pwd)/uploads:/app/dist/uploads spireguide
```

### 方式三：PM2 守护进程部署

```bash
npm install -g pm2
pm2 start 06-交付/build/dist/index.js --name spireguide
pm2 save
pm2 startup
```

## 目录结构（运行时）

```
spireguide/
├── dist/              # 编译后的后端代码
│   ├── index.js       # 入口
│   ├── routes/        # API路由
│   ├── database/      # 数据库连接
│   ├── public/        # 前端构建产物
│   └── data/          # SQLite数据库
│       └── spireguide.db
├── uploads/           # 用户截图上传目录
├── package.json
└── start.sh
```

## 配置项

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `PORT` | 3001 | 服务端口 |
| `NODE_ENV` | development | 环境模式 |
| `KIMI_API_KEY` | - | Kimi API密钥（接入真实AI时必填） |
| `RATE_LIMIT_WINDOW` | 900000 | 通用限流窗口(ms) |
| `RATE_LIMIT_MAX` | 100 | 通用限流次数 |
| `AI_RATE_LIMIT_WINDOW` | 3600000 | AI限流窗口(ms) |
| `AI_RATE_LIMIT_MAX` | 30 | AI限流次数 |

## 反向代理配置（Nginx）

```nginx
server {
    listen 80;
    server_name spireguide.example.com;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 上传文件大小限制
    client_max_body_size 10M;
}
```

## 数据备份

SQLite 数据库为单文件，备份简单：

```bash
# 热备份（SQLite WAL模式下可在线复制）
cp data/spireguide.db data/spireguide.db.$(date +%Y%m%d)
```

## 升级流程

1. 备份数据库和 uploads 目录
2. 替换 dist/ 和 public/ 目录
3. 如 Schema 有变更，运行数据库迁移脚本（后续版本提供）
4. 重启服务

## 已知部署限制

- 当前为单进程部署，未做集群适配
- 图片存储为本地文件系统，多节点部署需改对象存储
- AI功能当前为Mock，接入真实Kimi API需配置密钥
